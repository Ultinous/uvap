Configuring Multi Task Runner API (prototype)

This microservice does not require GPU resources.

The Multi Task Runner API microservice publishes an REST API through which can be accessed the Multi Task Runner functionality. The Multi Task Runner is a Java based project witch manage the different task instances.

Environment Variables

  • UVAP_MULTI_TASK_RUNNER_API_PROPERTY_FILE_PATHS: property files path list.

Multi Task Runner API reads its configuration from the property files in the above list during startup. See the list of properties below.

DescriptionPostfix of all identifiers for the Kafka stream processing applications.

RequiredRequired
Propertycom.ultinous.uvap.mtr.api.cache.max.bytes.buffering
DescriptionMaximum number of memory bytes to be used for buffering across all threads.
RequiredRequired
Propertycom.ultinous.uvap.mtr.api.commit.interval.ms
DescriptionThe frequency with which to save the position of the processor.
RequiredRequired
Propertycom.ultinous.uvap.mtr.api.max.task.idle.ms
DescriptionMaximum amount of time a stream task will stay idle when not all of its partition buffers contain records, to avoid potential out-of-order record processing across multiple input streams.
RequiredRequired
Propertycom.ultinous.uvap.mtr.api.buffered.records.per.partition
DescriptionMaximum number of records to buffer per partition.
RequiredRequired
Propertycom.ultinous.uvap.mtr.api.max.poll.records
DescriptionThe maximum number of records returned in a single Kafka call to poll.
RequiredRequired
Propertycom.ultinous.uvap.mtr.api.num.stream.threads
DescriptionThe number of threads to execute Kafka stream processing.
RequiredRequired
Propertycom.ultinous.uvap.mtr.api.state.dir
DescriptionDirectory location for Kafka state store.
RequiredRequired

Example properties file

com.ultinous.uvap.mtr.api.host=0.0.0.0
com.ultinous.uvap.mtr.api.port=4444
com.ultinous.uvap.mtr.api.jmx.port=4441
com.ultinous.uvap.mtr.api.monitoring.port=4442

com.ultinous.uvap.mtr.api.application.id.prefix=DEV_UMTR

com.ultinous.uvap.mtr.api.bootstrap.servers=kafka:9092
com.ultinous.uvap.mtr.api.cache.max.bytes.buffering=0
com.ultinous.uvap.mtr.api.commit.interval.ms=100
com.ultinous.uvap.mtr.api.max.task.idle.ms=2000
com.ultinous.uvap.mtr.api.buffered.records.per.partition=500
com.ultinous.uvap.mtr.api.max.poll.records=100
com.ultinous.uvap.mtr.api.num.stream.threads=8
com.ultinous.uvap.mtr.api.state.dir=/tmp/pycharm/kafka-state

com.ultinous.uvap.mtr.api.task.storage.class=com.ultinous.uvap.multi.task.runner.storage.KafkaStorage
com.ultinous.uvap.mtr.api.kafka.config.storage.topic=DEV_UMTR.permanent.storage

com.ultinous.uvap.mtr.api.task.handler.implementations=com.ultinous.uvap.mtr.task.crowddetection.CrowdDetectionTaskHandler, com.ultinous.uvap.task.kafka.empty.area.EmptyAreaTaskHandler,com.ultinous.uvap.task.kafka.wrong.direction.WrongDirectionTaskHandler

com.ultinous.uvap.mtr.api.service.implementations=com.ultinous.uvap.mtr.service.KafkaHeartbeatService
com.ultinous.uvap.mtr.api.service.heartbeat.output.topic=DEV_UMTR.heartbeat
com.ultinous.uvap.mtr.api.service.heartbeat.frequency.ms=10000
